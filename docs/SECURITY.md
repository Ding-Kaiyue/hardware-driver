# 安全政策

## 支持的版本

我们致力于为以下版本提供安全更新：

| 版本 | 支持状态 |
| ---- | -------- |
| 1.0.x | ✅ 支持 |

## 报告漏洞

如果您发现了安全漏洞，请通过以下方式联系我们：

- **GitHub Issues**: [提交安全漏洞报告](https://github.com/Ding-Kaiyue/hardware-driver/issues/new?template=security_report.md) - 提交严重安全漏洞报告，我们会尽快处理
- **Email**: kaiyue.ding@raysense.com (紧急情况)
- **Wechat**: d18292819833 (紧急情况)

### 报告内容

请包含以下信息：
- 漏洞的详细描述
- 重现步骤
- 可能的影响
- 建议的修复方案（如果有）

### 响应时间

我们承诺：
- 在收到报告后48小时内确认
- 在确认后7天内评估漏洞
- 在评估后30天内发布修复版本

## 安全最佳实践

### 使用建议
- 定期更新到最新版本
- 在生产环境中使用前充分测试
- 遵循最小权限原则
- 监控系统日志

### 开发建议
- 使用最新的依赖库版本
- 定期进行安全审计
- 遵循安全编码规范
- 进行代码安全审查
